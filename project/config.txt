# WIDTH represents the amount of squares from left to right.
# HEIGHT represents the amount of squares from top to bottom.
# ENTRY represents the coordinate for the start point for our maze.
# EXIT represents the coordinate where the "player" needs to go to solve the maze.
# SEED is the number that determines the whole output of the pathfinding and creating the maze. Keep it the same, output is the same, change it and the output will be completely different.
# OUTPUT_FILE is the name of where the maze will log it's output, a hexadecimal format map, the entry and exit coordinates and the shortest path from the entry to the exit.
# PERFECT is a boolean, representing if the maze will have only one possible solution (True) or more possible solutions (False).
# MUSIC_FILE is a boolean, do you want music or no!?
# COLOR_WALLS is a hexadecimal color value for the walls of the maze in the order of ARGB: 0xAARRGGBB
# COLOR_BACKGROUND is a hexadecimal color value of the background of the maze in the order of ARGB: 0xAARRGGBB
# COLOR_FOURTYTWO is a hexadecimal color value for the 42 blocks in the middle of the maze in the order of ARGB: 0xAARRGGBB
# COLOR_ENTRY is a hexadecimal color value for the entry of the maze in the order of ARGB: 0xAARRGGBB
# COLOR_EXIT is a hexadecimal color value for the exit of the maze in the order of ARGB: 0xAARRGGBB
# COLOR_MENUTEXT is a hexadecimal color value for the text in the options menu that clarifies what buttons you can press, in the order of BGR 0xBBGGRR.
# CELL_SIZE is the size of each individual cell used in multiple calculations for rendering and getting the size of the screen needed.
# PERC_WALL is size of a wall by multiplication of the cell size (probably best to keep it smaller than 1)
# PERC_PADDING is the size of the padding between the wall and the inside the cells. (probably best to keep it smaller than 1)

# All values from MUSIC_FILE downwards have defaults to fall back on, comment them to use the defaults.
